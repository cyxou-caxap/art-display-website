### 2.	Спецификация вариантов использования
<h4 align="right">Таблица 1 — ВИ «Оставить комментарий»</h4>

| **Идентификатор ВИ** | **UC-5** |
| :-------- | :------- |
| Наименование | Оставить комментарий |
| Автор | Гришкина Полина |
| Дата создания | 02.05.2024 |
| Основное действующее лицо | Пользователь |
| Дополнительное действующее лицо | - |
| Описание | Пользователь обращается к сайту, просматривает ленту постов, выбирает пост, который вызывает у него бурю эмоций, сдерживать которые невозможно, вследствие чего он решается оставить комментарий и, возможно, даже не один. |
| Приоритет | Средний |
| Условие-триггер | Пользователь выражает намерение поделиться мнением о работе посредством взаимодействия с элементом интерфейса «Оставить комментарий» |
| Предварительные условия | **PRE-1** Пользователь выполнил вход в систему (в противном случае будет доступен только просмотр комментариев от других пользователей).<br>**PRE-2** Автор поста разрешил его комментирование. |
| Выходные условия | **POST-1** Комментарий сохранён в БД с идентифицирующим его постом <br> **POST-2** Обновленный список комментариев отображается под постом <br> **POST-3** Система отправляет уведомление автору о новом мнение насчет его работы |
| Основные потоки | **1.0** Оставить комментарий <br> 1. Пользователь просматривает до конца пост <br> 2. Система выводит пользователю комментарии других пользователей (Е2) <br>3. Около списка комментариев отображается элемент интерфейса «Оставить комментарий или редактировать (если их количество >=1)» (см.1.1), (Е1) <br>4. Пользователь взаимодействует с данным элементом, выбирая оставить комментарий. <br> 5. Система предоставляет пользователю инструмент для написания отзыва <br>6. Пользователь записывает своё впечатление о работе <br>7. Пользователь ставит оценку посту <br>8. Пользователь нажимает на кнопку «Отправить» <br>9. Система добавляет отзыв пользователя в базу данных <br>10. Пользователь видит свой комментарий среди прочих <br>11. Система отправляет автору поста уведомление о новом комментарии под его работой |
| Альтернативные потоки | **1.1**	Редактировать комментарий (доступно в течение 10 минут)<br>1. Около оставленного пользователем комментария отображается элемент интерфейса «Редактировать комментарий» <br>2. Пользователь взаимодействует с этим элементом<br>3. Возврат к пункту 6 основного потока |
| Исключения | **Е1** Пользователь заблокирован модерацией <br>1.1 Пользователь не может оставлять отзывы <br>1.2 У пользователя не отображается элемент интерфейса «Оставить комментарий» <br>1.3 Завершение ВИ <br><br>**Е2** Автор поста запретил комментирование <br> 1.1 Пользователь не может оставлять и просматривать отзывы <br>1.2 У пользователя не отображается элемент интерфейса «Оставить комментарий» <br>1.3 Завершение ВИ|
| Бизнес-правила | В рамках данной работы не заданы |
| Другая информация | Пользователь имеет возможность отредактировать или удалить свой комментарий |
| Предположения | - |

<h4 align="right">Таблица 2 — ВИ «Добавить работу»</h4>

| **Идентификатор ВИ** | **UC-14** |
| :-------- | :------- |
| Наименование | Добавить работу |
| Автор | Гришкина Полина |
| Дата создания | 03.05.2024 |
| Основное действующее лицо | Пользователь |
| Дополнительное действующее лицо | - |
| Описание | Пользователь обращается к сайту с целью добавления работы |
| Приоритет | Высокий |
| Условие-триггер | Пользователь выражает намерение поделиться своей работой с другими людьми посредством взаимодействия с элементом интерфейса «Добавить работу» |
| Предварительные условия | **PRE-1** Пользователь выполнил вход в систему. |
| Выходные условия | **POST-1** Работа сохранена в БД с идентифицирующим её тегом (если выбран) и галерей <br>**POST-2** Обновленный список работ отображается в профиле <br>**POST-3** Система отправляет уведомление друзьям пользователя (если они согласились получать их) о его новой работе |
| Основные потоки | **1.0** Добавить работу<br>1. Пользователь переходит в свой профиль<br>2. На странице отображается элемент интерфейса «Добавить работу» <br>3. Пользователь взаимодействует с данным элементом<br>4. Система предлагает пользователю способ добавления работы(drag and drop/проводник) (см. 1.1)<br>5. Пользователь перетаскивает выбранный файл в предоставленное окно<br>6. Система проверяет файл на корректность (Е1, Е2)<br>7. Пользователь выбирает галерею (если данный шаг пропускается, ставится галерея по умолчанию: "Общая") (см. 1.2)<br>8. Пользователю предлагается указать тег (по умолчанию тег отсутствует) (см. 1.3)<br>9. Пользователь добавляет к работе название и описание (по умолчанию поля пусты) (см. 1.4)<br>10. Пользователь нажимает на кнопку «Опубликовать работу» <br>11. Система добавляет работу пользователя в базу данных<br>12. Пользователь видит свою работу среди прочих в галерее<br>13. Система отправляет друзьям пользователя уведомление о его новой работе |
| Альтернативные потоки | **1.1** Добавление через проводник<br>1. Пользователь взаимодействует с элементом "Выбрать файл из проводника"<br>2. Пользователь выбирает файл с устройства<br>3. Возврат к пункту 6 основного потока<br><br>**1.2** Выбрать галерею<br>1. Пользователь взаимодействует с элементом "Выбрать галерею"<br>2. Пользователь перенаправляется на страницу с галереями<br>3. Система предоставляет выбор: создать галерею/выбрать существующую. <br>4. Пользователь выбирает новую (задаёт ей имя) либо уже существующую галерею<br>5. Возврат к пункту 8 основного потока<br><br>**1.3** Добавление тега<br>1. Пользователь взаимодействует с элементом "Добавить тег"<br>2. Пользователь выбирает тег из выпадающего списка <br>3. Возврат к пункту 9 основного потока<br><br>**1.4** Добавить наименование и описание <br>1. Пользователь взаимодействует с элементом "Добавить наименование и описание"<br>2. Пользователь вводит до 64 символов в окно с названием работы и до 256 - в окно с описанием<br>3. Возврат к пункту 10 основного потока |
| Исключения | **Е1** Пользователь загрузил файл с недопустимым расширением<br>1. Система выводит сообщение о некорректном расширении файла<br>2. Происходит сброс выбранного файла<br>3. Возврат к пункту 4 основного потока<br><br>**Е2** Пользователь загрузил файл размером более 25 Мб<br>1. Система выводит сообщение о превышении ограничения на размер файла<br>2. Происходит сброс выбранного файла<br>3. Возврат к пункту 4 основного потока |
| Бизнес-правила | В рамках данной работы не заданы |
| Другая информация | - |
| Предположения | - |
